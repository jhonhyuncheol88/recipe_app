## 🛠 주요 변경 구성 항목

### 1. Domain Layer (핵심 비즈니스 로직)
- `lib/domain/entities/ingredient.dart`: 순수 Dart 객체로 정의된 식자재 엔티티.
- `lib/domain/repositories/ingredient_repository.dart`: 데이터 소스에 관계없는 추상 리포지토리 인터페이스.
- `lib/domain/usecases/batch_update_ingredients_usecase.dart`: 일괄 업데이트 및 삭제 로직을 캡슐화한 UseCase.

### 2. Data Layer (데이터 영속성)
- `lib/data/models/ingredient_model.dart`: JSON 및 SQLite 매핑 로직을 포함한 데이터 모델.
- `lib/data/repositories/ingredient_repository_impl.dart`: `sqflite`를 사용한 실제 데이터베이스 연동 구현체.

### 3. Presentation Layer (사용자 인터페이스)
- `lib/presentation/bloc/batch_edit/`: 일괄 수정을 위한 이벤트, 상태, BLoC 클래스.
- `lib/presentation/pages/batch_edit/`: BLoC 기반의 일괄 수정 화면.
- `lib/presentation/pages/batch_edit/widgets/ingredient_edit_item.dart`: 파일 크기 최적화를 위해 분리된 개별 수정 항목 위젯 (1,500줄 제한 준수).

### 4. Routing & CI
- `lib/router/app_router.dart`: 새로운 일괄 수정 페이지 라우트 등록 (`/ingredient/batch-edit`).
- `lib/screen/pages/ingredient/ingredient_main_page.dart`: 일괄 수정 화면으로 이동하는 버튼 추가.

## 📐 준수 사항 확인
- **OOP 원칙**: 인터페이스 기반의 리포지토리 설계 및 의존성 주입 구조 적용.
- **파일 크기**: 모든 신규 파일은 1,500줄 미만으로 유지됨.

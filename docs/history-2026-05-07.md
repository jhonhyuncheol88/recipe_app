## 1. 레시피 상세 공유 — 현재 배수 데이터 포함

### 동기
- 사용자 요구: 레시피 상세에서 공유를 누르면 단순히 이름/원가/판매가/마진율
  뿐 아니라, 화면에 떠 있는 **배수 슬라이더 값** 으로 스케일된 재료/소스
  목록까지 같이 공유되도록. 다른 사람에게 "이 배수로 만들면 재료가 얼마"
  라는 정보를 그대로 전달하기 위함.

### 변경
- **MODIFY** `lib/screen/pages/recipe/recipe_detail_page.dart`
  - `_share(...)` 시그니처에 `List<Ingredient> ingredients`, `List<Sauce> sauces`
    파라미터 추가. 호출부는 build 컨텍스트에서 watch 중인 `IngredientCubit`/
    `SauceCubit` state 를 그대로 넘김.
  - 공유 텍스트에 다음을 추가:
    - 빈 줄
    - `${배수}: N배 (스케일된 산출량${recipe.outputUnit})`
    - 빈 줄 + `[재료]` 헤더 + 각 재료 `- 이름: 양 × 배수 + unitId`
    - 빈 줄 + `[소스]` 헤더 + 각 소스 동일 포맷
  - `_formatShareQty(double qty, NumberFormatStyle style)` 헬퍼 추가 — 정수면
    `NumberFormatter.formatNumber`, 아니면 `toStringAsFixed(1)`.
  - 재료/소스가 둘 다 비어 있으면 배수 섹션 자체를 추가하지 않음 (짧은
    레시피의 공유 텍스트가 비어 보이지 않게).

### 결과
- 배수가 1 일 때도 재료/소스가 있으면 본인이 등록한 양이 그대로 공유됨.
- 배수 슬라이더를 5배로 올리고 공유하면 모든 재료/소스 양이 5배로 곱해진
  값으로 텍스트에 들어감.

## 2. 온보딩 페이지 재구성 — 광고 안내 + 프리미엄 유도

## 3. iOS Google 로그인 — 매번 계정 선택 강제

### 동기
- 사용자 보고: iPhone 에서 Google 로그인 누르면 계정 선택 화면이 안 뜨고
  Safari 시트에서 빈 화면 / "Continue as ○○○" 만 노출. (스크린샷 첨부)
- 진단:
  - `pubspec.lock` `google_sign_in_ios 5.9.0` 은 내부적으로 **AppAuth** 기반
    (`Podfile.lock` `google_sign_in_ios → AppAuth`). GoogleSignIn 8.x 네이티브
    SDK 가 아닌 OAuth 웹 흐름 사용.
  - AppAuth 흐름이 ASWebAuthenticationSession / SFSafariViewController 를
    띄우고 Safari 쿠키를 공유 → Safari 에 Google 계정 1개만 로그인된
    사용자는 picker 없이 "Continue as X" 만 봄.
  - 패키지 자체가 `prompt=select_account` 를 노출하지 않아 강제 불가.
- **iOS 설정 자체는 정상** — GoogleService-Info.plist 존재, `Info.plist:38`
  `GIDClientID` + `Info.plist:32-35` `CFBundleURLSchemes` 의 REVERSED_CLIENT_ID
  모두 일치.

### 변경
- **MODIFY** `lib/data/auth_repository.dart`
  - import 추가: `dart:io` `Platform`, `package:flutter/foundation.dart` `kIsWeb`.
  - `signInWithGoogle()` 분기:
    - **iOS (`!kIsWeb && Platform.isIOS`)** — `google_sign_in` 우회. Firebase
      Auth `signInWithProvider` + `GoogleAuthProvider..setCustomParameters({'prompt': 'select_account'})`
      + `addScope('email')` / `addScope('profile')`. OAuth 서버에 매번 picker
      를 요청하므로 Safari 쿠키와 무관하게 계정 선택 화면이 뜸. AppAuth 흐름
      이 아니라 Firebase OAuth 흐름으로 바뀌면서 빈 페이지 이슈도 회피.
    - **Android (그 외)** — `_googleSignIn.signOut()` 을 먼저 호출해 캐시된
      계정을 비운 뒤 `_googleSignIn.signIn()`. 기존 idToken (`_googleWebClientId`
      서버 클라이언트 ID 사용) 흐름 그대로 유지.
  - 취소 처리: iOS 의 `FirebaseAuthException` 코드 `web-context-canceled` /
    `canceled` / `user-cancelled` 중 하나면 기존 `AuthCancelledException` 매핑.
  - 기존 `signOut()` / `deleteAccount()` 의 `_googleSignIn.signOut()` 호출은
    Android 쪽 캐시 정리 용도로 그대로 둠 (iOS 에서는 사실상 no-op).

### 외부 설정 추가 작업 — 없음
- `signInWithProvider` 가 사용하는 OAuth callback URL scheme 은 Firebase 의
  REVERSED_CLIENT_ID 기반이고, 이미 `Info.plist` 에 등록돼 있어 추가 설정
  필요 없음.

### 결과 / 검증 포인트
- 코드 변경 분 `flutter analyze` clean.
- 사용자 실기기 테스트 필요 — iPhone 에서 Google 로그인 시:
  1. ASWebAuthenticationSession 시트가 뜨고
  2. Google 계정 선택 화면이 매번 노출
- **만약 빈 화면이 끝까지 빈 화면으로 남으면** Firebase Console → Authentication
  → Sign-in method → Google provider 의 iOS OAuth client redirect URI 등록
  + Authorized domains (`<project>.firebaseapp.com`) 점검 필요.

### 알아둘 것 (다음 세션용)
- iOS 만 `signInWithProvider` 로 분기됐다는 사실 자체가 이후 트러블슈팅의
  context. Android 회귀 테스트는 별도로 — `_googleSignIn` 흐름은 거의
  동일하지만 매 로그인마다 picker 강제됨.
- 계정 선택 강제가 부담스럽다는 피드백이 나오면 `signOut()` 호출 제거 +
  iOS 분기는 그대로 유지해도 OK (iOS 만 picker 강제로 차별화).
- google_sign_in 7.x 마이그레이션 시 이 우회를 되돌릴 수 있음 — 7.x 는
  GIDSignIn 8.x SDK 직접 사용 + `prompt` 파라미터 노출 가능성 있음. 다만
  지금은 다른 이슈가 더 우선.
